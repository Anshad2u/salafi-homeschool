# Salafi Homeschool — Legacy to Next.js Conversion Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Convert the legacy vanilla JS homeschool tracker into a full-stack Next.js 14 App Router application with PostgreSQL (Prisma), NextAuth PIN-based authentication, Tailwind CSS, and the same mobile-first role-based UI.

**Architecture:** Next.js 14 App Router with Server Components for pages, Client Components for interactive views, API Route Handlers for CRUD, Prisma + PostgreSQL for persistence, NextAuth (JWT) for PIN-based auth. Each legacy view function becomes a dedicated page component. Shared UI primitives (topbar, navbar, PIN pad, cards, progress bars) are extracted into reusable components.

**Tech Stack:** Next.js 14, React 18, TypeScript, Prisma 5 + PostgreSQL, NextAuth 4 (Credentials/JWT), Tailwind CSS 3, zod for validation.

---

## Current State

The legacy app (`legacy/`) is a single-page HTML/JS/CSS app with:
- **5 views:** Login, Admin (dashboard/planning/reports/settings), Teacher (today/quran/skills/reading), Student (myday/stars)
- **State:** localStorage persistence with a flat state shape `{users, tasks, quran, mastery, books, settings}`
- **Auth:** PIN-based with session storage, no real backend
- **Data:** All client-side, seeded from `seedState()`

The existing Next.js skeleton has:
- `package.json` with deps (Next 14, Prisma, NextAuth, Tailwind, zod, bcryptjs)
- `prisma/schema.prisma` with 6 models (Family, Profile, Task, QuranProgress, SkillProgress, Book)
- `src/lib/prisma.ts` singleton
- `tailwind.config.ts` with role-based colors
- `.env.example` for DATABASE_URL + NEXTAUTH_SECRET
